@twa.js/init-data

The package which provides utilities and types to work with the Telegram Web Apps init data. You can learn more about init data in this documentation.

Installation​

  • npm
  • yarn
  • pnpm
npm i @twa.js/init-data
yarn add @twa.js/init-data
pnpm i @twa.js/init-data

Usage​

Parsing​

This library contains the function parseInitData which is able to extract init data information from query parameters. Here comes the example of usage:

import {parseInitData} from '@twa.js/init-data';

// Let's imagine, we have init data in a raw format like this. Web Apps are
// sending it in the exact same format.
const initData =
  'query_id=AAHdF6IQAAAAAN0XohDhrOrc' +
  '&user=%7B%22id%22%3A279058397%2C%22first_name%22%3A%22Vladislav%22%2C%22last_name%22%3A%22Kibenko%22%2C%22username%22%3A%22vdkfrost%22%2C%22language_code%22%3A%22ru%22%2C%22is_premium%22%3Atrue%7D' +
  '&auth_date=1662771648' +
  '&hash=c501b71e775f74ce10e377dea85a7ea24ecd640b223ea86dfe453e0eaed2e2b2';

// Extract init data.
console.log(parseInitData(initData));
// or
console.log(parseInitData(new URLSearchParams(initData)));

// Output:
// {
//   authDate: 2022-09-10T01:00:48.000Z,
//   hash: 'c501b71e775f74ce10e377dea85a7ea24ecd640b223ea86dfe453e0eaed2e2b2',
//   queryId: 'AAHdF6IQAAAAAN0XohDhrOrc',
//   user: {
//     id: 279058397,
//     firstName: 'Vladislav',
//     lastName: 'Kibenko',
//     username: 'vdkfrost',
//     languageCode: 'ru',
//     isPremium: true
//   }
// }

Function extracts required parameters and automatically validates their types. It will throw an error in case, some property has invalid type or value.

Validation​

To validate the init data signature, the function validate is used. It expects passing init data presented in raw format (search params) and throws an error in some cases.

import {validate} from '@twa.js/init-data/validate';

const secretToken = '5768337691:AAH5YkoiEuPk8-FZa32hStHTqXiLPtAEhx8';
const initData =
  'query_id=AAHdF6IQAAAAAN0XohDhrOrc' +
  '&user=%7B%22id%22%3A279058397%2C%22first_name%22%3A%22Vladislav%22%2C%22last_name%22%3A%22Kibenko%22%2C%22username%22%3A%22vdkfrost%22%2C%22language_code%22%3A%22ru%22%2C%22is_premium%22%3Atrue%7D' +
  '&auth_date=1662771648' +
  '&hash=c501b71e775f74ce10e377dea85a7ea24ecd640b223ea86dfe453e0eaed2e2b2';

validate(initData, secretToken);

Function will throw an error in one of these cases:

  • "auth_date" should present integer
  • "auth_date" is empty or not found
  • "hash" is empty or not found
  • Signature is invalid
  • Init data expired

By default, the function checks the init data expiration. The default expiration duration is 1 day (86_400 seconds). It is recommended to always check to init data expiration as long as it could be stolen, but still valid. To disable this feature, pass {expiresIn: 0} as the third argument.

предупреждение

As long as there is no reason to validate init data on the client side, you are able to use validation utilities only in Node. Importing @twa.js/init-data/validation will lead to error as long as browser has no such package as node:crypto.
