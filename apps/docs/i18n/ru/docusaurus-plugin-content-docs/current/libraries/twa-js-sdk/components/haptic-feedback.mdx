# `HapticFeedback`

Контролирует тактильный отклик. Этот компонент позволяет создавать различные события тактильного отклика, возникающие в процессе взаимодействия пользователя с приложением. Более подробно об этой функциональной возможности можно прочитать в [документации](../../../functionality/haptic-feedback.md).

## Инициализация

```typescript
import { HapticFeedback } from '@twa.js/sdk';

const haptic = new HapticFeedback('6.3');
```

## Уведомления

`HapticFeedback` поддерживает 3 типа событий тактильного отклика - `impactOccurred`, `notificationOccurred` and `selectionChanged`.

### `impactOccurred`

Метод, который сообщает о том, что произошло столкновение объектов. Приложение Telegram проиграет это событие в зависимости от переданного значения:

- `light`, сообщает о коллизии между малыми или легковесными компонентами UI
- `medium`, сообщает о коллизии между компонентами UI среднего размера или веса
- `heavy`, сообщает о коллизии между компонентами UI большого размера или веса
- `rigid`, сообщает о коллизии между жёсткими или негибкими компонентами UI
- `soft`, сообщает о коллизии между мягкими или гибкими компонентами UI

```typescript
haptic.impactOccurred('medium');
```

### `notificationOccurred`

Метод, который уведомляет пользователя о том, что действие выполнено успешно, провалено, или создало предупреждение. Приложение Telegram проиграет это событие в зависимости от переданного значения:

- `error`, сообщает об ошибке
- `success`, сообщает об успешном выполнении
- `warning`, сообщает о предупреждении

```typescript
haptic.notificationOccurred('success');
```

### `selectionChanged`

Метод, который сообщает, что пользователь изменил свой выбор.

```typescript
haptic.selectionChanged();
```
